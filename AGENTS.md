# AGENTS

## Cursor Cloud specific instructions

This is a standard Android (Gradle) project — see `README.md` (section *Sestavení*) for the canonical commands. Notes specific to running it in the Cursor Cloud VM:

### Toolchain
- Build with **JDK 17** (AGP 8.5.2 / Gradle 8.7). The VM also has JDK 21 installed and on `PATH`; do not let Gradle pick it. JDK 17 is pinned via `~/.gradle/gradle.properties` (`org.gradle.java.home=/usr/lib/jvm/java-17-openjdk-amd64`).
- Android SDK lives at `/home/ubuntu/android-sdk` (cmdline-tools, `platform-tools`, `platforms;android-34`, `build-tools;34.0.0`).
- `local.properties` (with `sdk.dir=/home/ubuntu/android-sdk`) is required for Gradle to find the SDK. It is untracked/machine-specific and is regenerated by the startup update script, so do not rely on it being committed.

### Build / run
- Debug APK: `./gradlew assembleDebug` → `app/build/outputs/apk/debug/rfid_go_gps_<verze>.apk`. This is exactly what CI (`.github/workflows/android.yml`) runs.
- There is **no live "run" target in the VM**: the APK is built for `armeabi-v7a`/`arm64-v8a` only (no x86), targets a physical **Chainway C5** UHF reader, and bundles native `.so` from `app/libs/DeviceAPI_*.aar`. It cannot be installed on a standard x86 Android emulator, and RFID read/write/GPS need real hardware. Verify changes by building and by exercising pure-Java logic directly (e.g. `app/.../epc/EpcModel.java`, the documented core logic) in a small JVM harness.

### Lint / test
- `./gradlew lintDebug` runs but currently reports pre-existing `NewApi` errors (e.g. `Map#computeIfAbsent` in `CsvStore.java`, min SDK 21). These are existing code issues, not environment problems; CI does not run lint.
- No automated unit/instrumentation tests exist in the repo (`app/src/test` / `app/src/androidTest` are absent).
